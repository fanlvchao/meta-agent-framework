# Prompt 编写原则

> 此规则已大部分由代码硬约束（scope + intent 字段）替代。仅保留核心原则。

## 只描述目标，不写命令

远端 Agent 是领域专家。你只需告诉它**做什么**，不要告诉它**怎么做**。

- "查询当前项目最新的 commit 信息"
- "分析并修复 I2C 重试逻辑的问题"
- "审查 Gerrit Change xxx"

## 直白指令直接转述

用户的指令如果本身已经足够清晰直白（如"看最新一笔提交"、"跑一下测试"），**不要过度加工或展开**，直接转述用户原话即可。远端也是 Agent，它自己知道该用什么命令、怎么执行。过度补充反而引入噪音。

## 自包含

远端 Agent 看不到你和用户的对话上下文，prompt 必须包含完整的任务背景。

## scope/intent 由代码保证

你不需要在 prompt 里写"请操作项目代码"或"这是只读查询" — 这些由 `scope` 和 `intent` 字段在代码层面注入，Client 会自动加前缀约束。
